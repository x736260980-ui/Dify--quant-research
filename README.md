# Dify--quant-research
本项目是一个结合了金融量化数据与大模型（LLM）分析能力的投研辅助工具。通过封装基于 AkShare 和 Pandas-TA 的量化计算引擎，为 Dify 等工作流平台提供标准的 API 节点，实现从底层数据拉取、技术因子计算到最终文本诊断的可视化全链路。

本工具的运行逻辑分为三个核心阶段：

1. 因子计算与接口开发

底层数据处理引擎，负责解决外部数据源的限流与计算问题。

操作逻辑：通过 AkShare 接口按需获取股票、ETF 或行业板块的历史日线数据。引入前置本地缓存机制（失效周期1小时），有效拦截高频重复请求，保障接口稳定性。

因子计算：利用 Pandas-TA 计算 MACD、RSI(14)、MFI(14)、布林带等核心技术因子，并由 Matplotlib 自动生成包含走势与布林带的趋势图表。



2. 插件封装与静态分发

将复杂的金融计算过程转化为大模型可以直接理解和使用的标准化服务。

操作逻辑：使用 FastAPI 将上述逻辑封装为单一 POST 接口，返回包含“自然语言诊断结论”、“数值因子字典”、“历史收盘价轨迹”的结构化 JSON 数据。

静态资源：生成的走势图表会自动保存在本地 static/images 目录下，并通过 FastAPI 的静态挂载功能对外提供访问 URL。

3. web信息检索与舆情补充

这里复用了Dify--trend-crawler项目里的fastapi，web信息和情绪分析配合真实数据，可以确保活得更精准的结论，使用方法以及步骤可参考：

https://github.com/x736260980-ui/Dify--trend-crawler

4. 交叉验证与分析输出

在 Dify 平台完成量化数据与大模型推理的融合。

操作逻辑：通过项目中提供的 openapi.yaml 文件，将量化计算接口作为自定义工具零代码接入 Dify 工作流。

功能目标：Agent 根据接口返回的 analysis_summary（技术面结论）和 chart_url（走势图），结合大模型自身的金融分析能力进行二次交叉验证，最终输出结构化、可视化的投研参考报告。
